# Task 1: 定义原生 question AgentEvent 契约

## 描述

为 OpenCode 原生提问的 asked、replied、rejected 三类事件建立独立 AgentEvent 类型和 schema 元数据，不再复用 permission 事件。运行时防御性校验由事件生产端和 QuestionHandler 承担，本任务不改变 AgentEvent 构造器的全局行为。

## 依赖

- 前置 task：无
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
| ---- | -------- | ---- |
| 修改 | `src/drivers/agent-driver.js` | 新增 question 事件常量和 DATA_SCHEMAS |
| 修改 | `test/agent-driver-schema.test.js` | 验证三类事件常量和 schema 字段契约 |

## 复杂度

low，原因：改动集中在事件常量和 schema 元数据，不涉及异步状态或全局构造验证。

## 实现步骤

- [ ] 先增加失败测试，固定事件字符串为 `question_asked`、`question_replied`、`question_rejected`。
- [ ] 为 asked 事件声明 `requestID`、`sessionID`、非空 `questions` 及题目字段的 schema；保留可选 `tool`、`multiple`、`custom`。
- [ ] 为 replied 事件声明 `requestID`、`sessionID` 和二维字符串数组 `answers` 的 schema。
- [ ] 为 rejected 事件声明 `requestID` 和 `sessionID` 的 schema。
- [ ] 不在 AgentEvent 构造器中增加全局运行时验证；无效 question 载荷的拒绝分别由 T3 和 T6 测试。
- [ ] 确认既有 permission schema 和其他 AgentEvent 不受影响。
