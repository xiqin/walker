# Task 4: 增加 Driver 原生 question 回复接口

## 描述

在 OpenCode Driver 中增加与 permission 分离的 `replyQuestion(agentRef, requestID, answers)`，并原样传播 Bridge 的结构化成功或失败结果。

## 依赖

- 前置 task：T2
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
| ---- | -------- | ---- |
| 修改 | `src/drivers/opencode-driver.js` | 新增独立 question 回复方法并调用 TUI Bridge |
| 修改 | `test/opencode-driver.test.js` | 验证路由、载荷、错误传播和 permission 回归 |

## 复杂度

medium，原因：代码量较小，但必须保持不同 transport 和 permission 语义隔离，并准确传播可靠性字段。

## 实现步骤

- [ ] 先增加失败测试，固定公开签名为 `replyQuestion(agentRef, requestID, answers)`。
- [ ] T2 完成后读取 Bridge 的实际公开方法和结构化错误，按其最终接口接线。
- [ ] 根据 `agentRef` 解析正确的 TUI session/runtime，并调用 `tuiBridge.replyQuestion(sessionRef, requestID, answers)`。
- [ ] 对不支持原生 question reply 的 transport 返回稳定且不可重试的结构化错误，不回退为 prompt 或 permission。
- [ ] 不包装丢失 Bridge 的 `code`、`deliveryPhase`、`sdkInvoked`、`safeToRetry` 字段。
- [ ] 测试 Bridge reject 的 Error 四个结构化属性在 Driver 调用端保持完全一致，包括 accepted 超时与 accepted 后不确定结果。
- [ ] 保持 `replyPermission` 的签名、调用目标和已有测试不变。
