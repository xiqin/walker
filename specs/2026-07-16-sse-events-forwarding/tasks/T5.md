# Task 5: 构建原生 question 飞书卡片

## 描述

新增与 permission 卡片分离的原生 question 卡片构建器，支持预设选择、自定义输入、问题序号、稳定回调标识、重试和全部用户可见终态。

## 依赖

- 前置 task：无
- 外部依赖：飞书卡片 2.0 表单组件

## 涉及文件

| 操作 | 文件路径 | 说明 |
| ---- | -------- | ---- |
| 修改 | `src/platform/feishu/cards.js` | 新增原生问题卡片和状态卡片构建器 |
| 修改 | `src/platform/feishu/commands.js` | 更新 `/answer` 帮助和 questionKey 动作契约 |
| 修改 | `test/feishu-cards.test.js` | 验证表单结构、动作值和状态展示 |
| 修改 | `test/feishu-commands.test.js` | 验证新 `/answer` 帮助与参数解析 |

## 复杂度

medium，原因：需组合单选、多选、输入和多种状态卡片，但不负责业务状态转换。

## 实现步骤

- [ ] 先增加失败测试，固定导出 `buildNativeQuestionCard(options)` 和 `buildNativeQuestionStatusCard(options)`。
- [ ] `buildNativeQuestionCard` 展示 header、question、选项 label/description 和“问题 i/N”。
- [ ] 单选使用 `select_static`，多选使用 `multi_select_static`，字段名为 `question_selected`，选项值为 `option_0`、`option_1` 等稳定序号。
- [ ] `custom !== false` 时渲染字段名为 `question_custom` 的 input；无选项且允许 custom 时仅显示 input。
- [ ] 提交动作生成 `cmd:/answer {questionKey} --form {walkerSessionId}`，并携带 routeKey；questionKey 使用 `requestID:questionIndex`。
- [ ] 状态构建器覆盖 preparing、answered、submitting、replied、rejected、processed_unknown、feishu_unavailable、expired 和 retryable；重试动作使用 `--retry` 且不携带表单值。
- [ ] 更新 `/answer` 命令帮助为 `/answer <questionKey> --form <walkerSessionId>` 与 `/answer <questionKey> --retry <walkerSessionId>`，移除旧的手工 `<questionId> <value>` 产品契约。
- [ ] 保留通用卡片辅助函数和现有非 question 卡片行为，不修改事件解析协议。
