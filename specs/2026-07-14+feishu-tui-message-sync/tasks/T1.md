# Task 1: 修复飞书线程消息到已 attach 会话的入站路由

## 描述

验证并完成 thread route 未绑定时向同群根 route 的受限回退，使飞书消息能投递到已 attach 的 OpenCode session，同时不跨群、不改变已绑定线程的优先级。

## 依赖

- 前置 task：无
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/dispatch/message-dispatcher.js` | 在严格条件下回退到同群根 route |
| 修改 | `test/message-dispatcher.test.js` | 覆盖回退成功、线程优先和未绑定行为 |

## 复杂度

medium，原因：改动局部，但路由错误可能导致跨会话投递，必须验证回退边界和现有去重、touchRoute 语义。

## 实现步骤

- [ ] 审阅工作区已有 dispatcher 差异，确认不覆盖其他并行改动。
- [ ] 使用回归测试对 HEAD 旧实现建立失败信号：thread 消息 route 未绑定、同群根 route 已 attach 时，消息应进入该 session。
- [ ] 保证仅在 `routeMode === 'thread'`、事件包含 `rootId` 和 `chatId`、原 route 未绑定时构造同群根 route 并查询。
- [ ] 保证线程 route 已绑定时不回退，不同群 route 不参与匹配，两个 route 都未绑定时仍发送原有引导卡片。
- [ ] 确认 driver prompt、`touchRoute` 和 watcher 使用最终命中的 route/session。
