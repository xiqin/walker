# Task 3: Session 删除与终态保护

## 描述

收紧 session 服务的删除语义，防止 deleted session 被重新绑定、脏 route 命中，并保护 stopped/deleted 终态不被后续状态更新覆盖。

## 依赖

- 前置 task：T1
- 外部依赖：Node.js `node:test`

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/core/session-service.js` | `getCurrent()` 过滤 deleted 并清理脏 route；`bindRoute()` 拒绝 deleted；状态更新保护 stopped/deleted |
| 修改 | `test/session-service.test.js` | 增加删除后绑定、脏 route、终态覆盖回归测试 |

## 复杂度

medium，原因：状态机逻辑集中在单个服务，但会影响 dispatch 和 driver 完成回调。

## 实现步骤

- [ ] 步骤 1：新增测试，删除 session 后 `bindRoute(route, id)` 抛错。
- [ ] 步骤 2：新增测试，route 持久化指向 deleted session 时 `getCurrent(route)` 返回 `null` 并移除该 route。
- [ ] 步骤 3：新增测试，`stopSession()` 后调用 `markIdle()` 或 `markError()` 不覆盖 stopped；`deleteSession()` 后任何状态更新不覆盖 deleted。
- [ ] 步骤 4：实现 `bindRoute()` 和 `getCurrent()` deleted 防护。
- [ ] 步骤 5：在 `_updateState()` 中保护终态，必要时返回当前 session 而不写入新状态。
