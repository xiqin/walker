# Task 1: Core store 与 HTTP/SSE 边界

## 描述

修复 `JsonStore` 默认值引用污染，并让普通 HTTP 请求和 SSE 连接具备明确的超时、响应校验与标准事件分帧解析。

## 依赖

- 前置 task：无
- 外部依赖：Node.js 内置 `http`、`https`、`node:test`

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/core/json-store.js` | 读取 fallback 时返回默认值深拷贝，避免 `update()` 污染构造默认值 |
| 修改 | `src/core/http-helper.js` | 为 `httpRequest()` 增加超时选项；为 `sseConnect()` 增加状态码、`Content-Type` 校验与空行分帧解析 |
| 修改 | `test/json-store.test.js` | 增加文件缺失和损坏后的 fallback 不被历史更新污染的测试 |
| 修改 | `test/http-helper.test.js` | 增加 HTTP 超时、SSE 非成功响应、非 SSE 响应、多行 data 和 keepalive 的测试 |

## 复杂度

high，原因：`http-helper` 是 driver 共享边界，错误处理和事件解析会影响后续 OpenCode driver 行为。

## 实现步骤

- [ ] 步骤 1：为 `JsonStore` 增加默认值克隆测试，覆盖文件不存在、文件损坏、`update()` 原地变异三种场景。
- [ ] 步骤 2：实现默认值克隆，优先使用 JSON 可序列化深拷贝，保持现有 store 数据结构不变。
- [ ] 步骤 3：为 `httpRequest()` 增加 `timeoutMs` 支持，并在超时时销毁请求且返回带 URL 的错误。
- [ ] 步骤 4：让 `sseConnect()` 在 `statusCode` 非 2xx 或 `Content-Type` 不是 `text/event-stream` 时 reject，且不调用成功握手回调。
- [ ] 步骤 5：按 SSE 空行分帧组装事件，支持多行 `data:`、忽略注释行和 `id:`/`event:` 元数据。
