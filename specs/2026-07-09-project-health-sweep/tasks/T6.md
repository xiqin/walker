# Task 6: Windows/WSL terminal 命令转义与完整验证

## 描述

修复 Windows 和 WSL 打开交互终端时的 `cmd.exe /k` 参数转义，并在全部修复完成后运行完整检查。

## 依赖

- 前置 task：T1, T2, T3, T4, T5
- 外部依赖：Windows `cmd.exe` 参数规则、Node.js `node:test`

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/runtime/windows-runtime.js` | 为 `cmd.exe /k` 命令片段提供安全转义，覆盖 `& | < > ^ % ! "` 等字符 |
| 修改 | `src/runtime/wsl-runtime.js` | 对 WSL distro、命令和参数进行 Windows shell 转义，并保留 WSL 内参数边界 |
| 修改 | `test/runtime.test.js` | 增加 Windows/WSL openTerminal 注入字符和引号测试 |
| 创建 | `specs/2026-07-09-project-health-sweep/verify-report.md` | 记录最终 `npm run check` 和关键局部测试结果 |

## 复杂度

medium，原因：改动集中在 runtime，但 Windows shell 字符转义需要覆盖多个特殊字符组合。

## 实现步骤

- [ ] 步骤 1：新增测试，Windows `openTerminal()` 参数含 `& | < > ^ % ! "` 时生成的命令字符串不会裸露 shell 控制符。
- [ ] 步骤 2：新增测试，WSL distro、cwd、server URL 和参数含引号或控制符时仍被完整包裹并转义。
- [ ] 步骤 3：实现 Windows command-line 转义辅助逻辑，保持 `spawn('cmd.exe', ['/k', commandString])` 调用形状不变。
- [ ] 步骤 4：实现 WSL openTerminal 的 distro 和 WSL 内参数转义，避免双层引号破坏命令。
- [ ] 步骤 5：运行 `node --test test/runtime.test.js`。
- [ ] 步骤 6：运行 `npm run check`，将命令和结果写入 `verify-report.md`。
