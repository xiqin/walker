# Task 5: OpenCode driver 响应校验与跨 session SSE 过滤

## 描述

收紧 OpenCode driver 的 session 创建响应校验，并让目录级 SSE 只接受明确属于目标 session 的关键事件。

## 依赖

- 前置 task：T1, T3
- 外部依赖：Node.js `node:test`

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/drivers/opencode-driver.js` | 校验 create session HTTP 状态和 session id；过滤无 session id 的文本与终止事件；必要时更新错误事件映射 |
| 修改 | `test/opencode-driver.test.js` | 增加 createSession 失败、缺 id、无 session id SSE 串流和提前终止测试 |

## 复杂度

high，原因：driver 同时处理 HTTP 创建、SSE 长连接和后台 watch，过滤规则必须避免丢失目标 session 输出。

## 实现步骤

- [ ] 步骤 1：新增测试，`createSession()` 收到非 2xx 响应时抛出诊断错误且不打开终端。
- [ ] 步骤 2：新增测试，`createSession()` 收到 2xx 但缺 session id 时抛出诊断错误。
- [ ] 步骤 3：新增测试，prompt SSE 中无 session id 的 assistant text 不进入目标输出。
- [ ] 步骤 4：新增测试，无 session id 的 `session.status idle` 不会提前结束目标 prompt。
- [ ] 步骤 5：实现 create session 响应校验，错误包含 status、serverUrl 和响应摘要。
- [ ] 步骤 6：调整 `_eventBelongsToSession()` 和 `_isTerminalSSEEvent()`，要求 session 级关键事件必须带匹配 id。
- [ ] 步骤 7：确认 watchSession 与 prompt 使用同一过滤语义，不把其他 session 事件转发到当前飞书会话。
