# Task 6: 消息调度、去重、附件与表情

## 描述

把飞书普通消息、Walker session、AgentDriver 和飞书 UI 串成完整消息调度链，并补齐去重、附件目录和 reaction/done emoji 行为。

## 依赖

- 前置 task：T2、T3、T4、T5
- 外部依赖：飞书附件下载/上传 API、飞书 reaction API、AgentDriver

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/core/message-dedup.js` | 5 分钟 message_id 去重窗口 |
| 创建 | `src/dispatch/message-dispatcher.js` | 普通消息投递、未绑定引导、session 排队、driver event 到飞书 UI |
| 创建 | `src/dispatch/attachment-service.js` | 飞书文件/图片下载到 session attachments 目录，driver 文件事件上传回飞书 |
| 创建 | `test/message-dedup.test.js` | 覆盖去重窗口和过期清理 |
| 创建 | `test/message-dispatcher.test.js` | 覆盖路由绑定、未绑定、driver event、进度卡片 |
| 创建 | `test/attachment-service.test.js` | 覆盖附件路径、安全文件名、入站/出站接口调用 |

## 复杂度

high，原因：这是跨平台、session、driver 和 UI 的主集成点，需要通过 mock 保证行为可验证。

## 实现步骤

- [ ] 步骤 1：实现 `MessageDedup`，按飞书 `messageId` 记录最近 5 分钟处理结果，重复消息直接返回已处理。
- [ ] 步骤 2：实现 `MessageDispatcher.handleIncomingMessage(event)`：构造 routeKey，处理去重，若未绑定则返回引导卡片，不自动投递给任意 session。
- [ ] 步骤 3：实现 `/new` 的调度路径：调用 driver `ensureReady/createSession`，再调用 session service 创建 Walker session 并自动绑定。
- [ ] 步骤 4：实现绑定 session 的 prompt 路径：标记 running，创建进度卡片，迭代 driver `prompt()` 的 AgentEvent 并更新卡片或回复文本，完成后标记 idle。
- [ ] 步骤 5：实现 session 忙碌策略：默认同 session 内串行排队；如果当前实现只支持单 turn，则返回忙碌提示并保持状态一致。
- [ ] 步骤 6：实现 `AttachmentService`：入站文件/图片保存到 `.walker/attachments/<walkerSessionId>/`，出站 `file`/`image` AgentEvent 通过飞书上传并发送。
- [ ] 步骤 7：实现 reaction/done emoji 调用开关；API 失败不影响主消息链，但记录日志。
