# Task 3: 飞书平台层与命令解析

## 描述

实现飞书长连接入口、事件解析、OpenAPI 文本回复封装和 Walker 命令解析执行，不接入卡片渲染细节。

## 依赖

- 前置 task：T1、T2
- 外部依赖：`@larksuiteoapi/node-sdk`、飞书 OpenAPI

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/platform/feishu/api.js` | tenant token、文本 reply、普通消息发送、错误封装 |
| 创建 | `src/platform/feishu/events.js` | 解析 `im.message.receive_v1` 与 `card.action.trigger` 事件 |
| 创建 | `src/platform/feishu/commands.js` | 解析 `/new`、`/list`、`/use`、`/current`、`/stop`、`/delete`、`/agents`、`/runtime`、`/help` |
| 创建 | `src/platform/feishu/platform.js` | 启动飞书 WSClient，派发文本事件与按钮事件到 command/dispatcher |
| 创建 | `test/feishu-events.test.js` | 覆盖飞书事件字段提取 |
| 创建 | `test/feishu-commands.test.js` | 覆盖命令解析与 session service 调用 |

## 复杂度

high，原因：需要隔离飞书 SDK、OpenAPI、命令行为和 session service，同时避免平台层依赖具体 AgentDriver。

## 实现步骤

- [ ] 步骤 1：实现 `FeishuApi`，支持 tenant token 缓存、`replyText(replyCtx, text)`、`sendText(chatId, text)`，错误包含 endpoint 和飞书返回码。
- [ ] 步骤 2：实现 `parseMessageEvent(data)`，提取 `chatId`、`messageId`、`rootId`、`parentId`、`openId`、`messageType`、`text`、`createTime`。
- [ ] 步骤 3：实现 `parseCardAction(data)`，提取 action value、chatId、messageId、openId、原始 form value。
- [ ] 步骤 4：实现 `parseCommand(text)` 和 `executeCommand(command, context)`，命令层只调用 session service 和后续注入的 driver dispatcher 接口。
- [ ] 步骤 5：`/help` 返回完整命令说明；`/current`、`/use`、`/use off`、`/delete` 走 session service；`/new` 先返回 create request 交给 dispatcher 完成 driver 创建。
- [ ] 步骤 6：实现 `FeishuPlatform.start()`，注册 `im.message.receive_v1` 和 `card.action.trigger`，把普通文本交给 dispatcher，把命令交给 command service。
