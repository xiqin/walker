# Task 5: Runtime 与 OpencodeDriver

## 描述

实现 AgentDriver 抽象、driver 注册表、Windows/WSL runtime，以及通过 `opencode serve` HTTP/SSE 控制 opencode session 的 P0 driver。

## 依赖

- 前置 task：T1、T2
- 外部依赖：本机 `opencode` 命令、opencode HTTP API、Node HTTP client、`child_process`

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/drivers/agent-driver.js` | 记录统一 AgentDriver 接口、AgentEvent 约定和错误类型 |
| 创建 | `src/drivers/driver-registry.js` | 注册并查找可用 driver |
| 创建 | `src/drivers/opencode-driver.js` | 检测/autostart opencode serve、创建 session、prompt、SSE 转 AgentEvent |
| 创建 | `src/drivers/stub-drivers.js` | Claude Code 与 Codex 的扩展点占位实现，返回明确未启用错误 |
| 创建 | `src/runtime/windows-runtime.js` | Windows 原生命令启动和诊断 |
| 创建 | `src/runtime/wsl-runtime.js` | WSL 命令启动、distro 检查、IP 探测 |
| 创建 | `src/runtime/runtime-factory.js` | 按配置选择 runtime |
| 创建 | `test/opencode-driver.test.js` | 使用 fake HTTP/SSE 覆盖 driver 行为 |
| 创建 | `test/runtime.test.js` | mock child_process 覆盖 Windows/WSL 命令构造与错误 |

## 复杂度

high，原因：需要把外部进程、HTTP API、SSE 事件和统一 AgentEvent 映射做成可测边界。

## 实现步骤

- [ ] 步骤 1：定义 `DriverRegistry`，内置 `opencode`、`claude`、`codex`，其中 `claude` 和 `codex` 返回未实现但可诊断的错误。
- [ ] 步骤 2：实现 `WindowsRuntime.spawn(command, args, options)` 和 `WslRuntime.spawn(command, args, options)`；WSL runtime 使用 `wsl.exe -d <distro> -- <command>`。
- [ ] 步骤 3：实现 WSL server URL 推导：先尝试配置的 `OPENCODE_SERVER_URL`，失败时调用 `hostname -I` 获取 WSL IP。
- [ ] 步骤 4：实现 `OpencodeDriver.ensureReady(config)`，检查 `/` 或健康端点；未启动且 `OPENCODE_SERVER_AUTOSTART=true` 时通过 runtime 启动 `opencode serve --hostname 127.0.0.1 --port <port>` 或 WSL 对应命令。
- [ ] 步骤 5：实现 `createSession`、`resumeSession`、`prompt`、`stop`、`delete`；opencode endpoint 以当前安装版本实测为准，测试中通过 fake client 固化 adapter 层输入输出。
- [ ] 步骤 6：实现 SSE 解析，把 opencode 事件映射为 `status`、`text`、`reasoning`、`tool_use`、`error`、`done` AgentEvent。
- [ ] 步骤 7：所有外部错误包含 agent、runtime、cwd、command 或 serverUrl。
