# Task 7: 入口替换、脚本与端到端启动

## 描述

把旧 `src/index.js` 的 opendray bridge adapter 替换为 Walker Hub 入口，串联配置、session store、driver registry、dispatcher 和 FeishuPlatform，并更新 PowerShell 后台脚本。

## 依赖

- 前置 task：T1、T2、T3、T4、T5、T6
- 外部依赖：飞书长连接、可选 opencode server

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/index.js` | 删除旧 opendray bridge 连接逻辑，改为调用 Walker bootstrap |
| 创建 | `src/app/bootstrap.js` | 组装配置、store、service、runtime、driver、dispatcher、platform |
| 修改 | `scripts/start.ps1` | 更新进程名、日志名和说明 |
| 修改 | `scripts/status.ps1` | 更新状态检测和输出文本 |
| 修改 | `scripts/logs.ps1` | 更新日志路径 |
| 修改 | `scripts/stop.ps1` | 更新停止逻辑和进程匹配 |
| 创建 | `test/bootstrap.test.js` | mock 所有外部依赖，验证启动组装和关闭流程 |

## 复杂度

high，原因：这是从旧 MVP 切换到新架构的实际入口，必须确保不再触碰 opendray 并保留启动可靠性。

## 实现步骤

- [ ] 步骤 1：实现 `createApp(config)`，返回 `{ start, stop }`，并注入 logger、session service、runtime factory、driver registry、dispatcher、FeishuPlatform。
- [ ] 步骤 2：重写 `src/index.js`，只负责加载配置、创建 app、注册 `SIGINT`/`SIGTERM`、输出启动日志和调用 `start()`。
- [ ] 步骤 3：删除入口中所有 `WebSocket` opendray bridge 连接、`OPENDRAY_*` 读取、`/select` 转发、opendray REST login/list 调用。
- [ ] 步骤 4：更新 scripts 下后台管理命令，使日志路径指向 `.walker/logs/` 或当前项目 `logs/` 的新 Walker adapter 日志。
- [ ] 步骤 5：实现 bootstrap 测试，确保 `start()` 调用 FeishuPlatform，不调用任何 opendray URL 或 bridge token。
- [ ] 步骤 6：执行无 opendray 环境启动冒烟；若缺少真实飞书凭据，记录为配置缺失错误而非 opendray 错误。
