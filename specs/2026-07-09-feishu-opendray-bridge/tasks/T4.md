# Task 4: 飞书卡片、按钮与进度 UI

## 描述

实现 `/list` 会话卡片、按钮 action 编码、Agent 进度卡片和 Patch 更新失败降级策略。

## 依赖

- 前置 task：T2、T3
- 外部依赖：飞书 interactive card 与 Patch API

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/platform/feishu/cards.js` | 渲染 session 列表、未绑定引导、错误提示和基础按钮 |
| 创建 | `src/platform/feishu/progress-card.js` | 管理一个 prompt 对应一张进度卡片，支持追加事件和定稿 |
| 创建 | `test/feishu-cards.test.js` | 覆盖卡片结构、按钮 value、状态显示 |
| 创建 | `test/progress-card.test.js` | 覆盖进度聚合、Patch 降级、新消息回退 |

## 复杂度

high，原因：飞书卡片 schema、按钮回调、状态展示和 Patch 失败降级需要共同稳定。

## 实现步骤

- [ ] 步骤 1：实现 `renderSessionListCard(sessions, currentSessionId)`，展示 id、title、agent、cwd、status、updatedAt，并包含绑定、继续、停止、删除按钮。
- [ ] 步骤 2：实现 `renderUnboundRouteCard(route)`，提示用户发送 `/new` 或通过 `/list` 选择已有 session。
- [ ] 步骤 3：实现按钮 value 编码规范，统一使用 `{ action: 'cmd:/use <id>' }`、`{ action: 'cmd:/stop <id>' }` 等结构。
- [ ] 步骤 4：实现 `ProgressCard`，把 `status`、`text`、`reasoning`、`tool_use`、`error`、`done` AgentEvent 聚合为飞书卡片元素。
- [ ] 步骤 5：为 `FeishuApi` 预留或补充 `sendCard`、`replyCard`、`patchCard` 方法调用点；如果 Patch 失败，返回降级发送新消息的指令。
- [ ] 步骤 6：限制卡片内容长度，过长文本折叠为摘要，避免飞书 payload 过大。
