# Task 1: 项目骨架、配置与持久化

## 描述

建立 Walker Hub 的基础模块：配置读取、ID 生成、日志和本地 JSON 持久化，并调整 `package.json` 的检查脚本以支持后续单元测试。

## 依赖

- 前置 task：无
- 外部依赖：Node.js、现有 `@larksuiteoapi/node-sdk`

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `package.json` | 更新项目名称、描述和 `check` 脚本，使语法检查与 `node:test` 可执行 |
| 创建 | `src/config/env.js` | 集中读取 `.env`、飞书、Walker、runtime、opencode 配置 |
| 创建 | `src/core/id.js` | 生成 `wks_` 等本地稳定 ID |
| 创建 | `src/core/json-store.js` | 提供原子读写 JSON、目录初始化和默认值合并 |
| 创建 | `src/core/logger.js` | 统一结构化日志输出，包含上下文和错误信息 |
| 创建 | `test/config-env.test.js` | 覆盖 `.env` 和环境变量解析 |
| 创建 | `test/json-store.test.js` | 覆盖 JSON store 的读写、缺省值和损坏文件错误 |

## 复杂度

medium，原因：需要先形成后续任务依赖的稳定基础接口，但逻辑以本地文件和纯函数为主。

## 实现步骤

- [ ] 步骤 1：定义 `loadEnvConfig(options)`，支持 `FEISHU_APP_ID`、`FEISHU_APP_SECRET`、`CC_CONNECT_CONFIG`、`FEISHU_ROUTE_MODE`、`WALKER_DATA_DIR`、`WALKER_DEFAULT_AGENT`、`WALKER_DEFAULT_RUNTIME`、`WALKER_DEFAULT_CWD`、`WALKER_WSL_DISTRO`、`OPENCODE_SERVER_URL`、`OPENCODE_SERVER_AUTOSTART`、`OPENCODE_CMD`、`OPENCODE_MODEL`、`OPENCODE_AGENT`、`FEISHU_PROGRESS_STYLE`、`FEISHU_REACTION_EMOJI`、`FEISHU_DONE_EMOJI`。
- [ ] 步骤 2：保留从 `~/.cc-connect/config.toml` 读取首个 `app_id` / `app_secret` 的兼容路径，但不读取任何 opendray 配置。
- [ ] 步骤 3：实现 `JsonStore`，负责创建 `.walker/`、`messages/`、`attachments/`、`logs/` 目录，并提供 `read(defaultValue)`、`write(value)`、`update(mutator)`。
- [ ] 步骤 4：实现 `createId(prefix)` 与 `createLogger(scope)`，日志错误对象必须包含 message、stack、cause 摘要。
- [ ] 步骤 5：修改 `package.json`，把 `check` 改成语法检查所有 `src/**/*.js` 并运行 `node --test test/*.test.js`。
- [ ] 步骤 6：补充单元测试，验证配置默认值、cc-connect 凭据读取、本地数据目录初始化和 JSON 损坏时报错可诊断。
