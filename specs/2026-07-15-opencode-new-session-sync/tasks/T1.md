# Task 1: 实现事件驱动的新会话重新绑定

## 描述

让生成的 OpenCode TUI bridge 插件在根会话创建或用户选择会话时更新活动会话并重新注册 Walker，同时保持事件和 delivery 的会话归属正确。

## 依赖

- 前置 task：无
- 外部依赖：OpenCode 1.17.20 TUI plugin 事件接口；不新增 npm 依赖

## 涉及文件

| 操作 | 文件路径 | 说明 |
| ---- | -------- | ---- |
| 修改 | `test/opencode-hook-installer.test.js` | 增加生成插件运行级会话切换测试，并更新模板版本断言 |
| 修改 | `src/opencode-hook/plugin-template.js` | 生成版本 2 插件，维护和使用事件驱动的活动会话 |

## 复杂度

medium，原因：只改两个文件，但生成字符串中的异步事件、轮询和 delivery 状态需要共同验证，避免滞后 route 与并发 tick 产生会话串线。

## 实现步骤

- [ ] 在生成插件运行测试中固定 `api.route.current` 为 `ses_old`，触发根级 `session.created(ses_new)`，证明插件会注册、轮询并向 `ses_new` 执行飞书 delivery。
- [ ] 增加 `tui.session.select` 和带 `parentID` 的 `session.created` 场景，分别验证已有根会话切换和子会话过滤。
- [ ] 在模板中增加 `activeSessionId`，仅在活动状态为空时从 route 初始化或补足；tick、register 和 delivery 均使用该状态。
- [ ] 注册 `session.created` 与 `tui.session.select` handler；有效切换立即更新活动会话，清除成功注册标记并触发 tick，注册失败时保留新活动会话供后续重试。
- [ ] 调整 idle/error 处理，始终按事件携带的 session ID 查找和上报对应 delivery，不把旧会话事件归到当前活动会话。
- [ ] 将模板标记升级为版本 2，并验证 installer 会覆盖版本 1、对完全匹配的版本 2 不重复写入。
