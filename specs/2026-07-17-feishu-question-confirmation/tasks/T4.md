# Task 4: 新增 buildQuestionCard 和 buildQuestionRepliedCard

## 描述

在 `src/platform/feishu/cards.js` 新增 `buildQuestionCard(questionEvent, sessionId, routeKey)` 和 `buildQuestionRepliedCard(questionId, answer)`，支持 4 种 inputMode（confirm/single_select/multi_select/text）的飞书卡片渲染，使用 `buildButtonValue` 生成 `cmd:/answer` 动作 value。

## 依赖

- 前置 task：无
- 外部依赖：飞书交互卡片协议（`multi_select_static`、`input` 组件）

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/platform/feishu/cards.js` | 新增 `buildQuestionCard`、`buildQuestionRepliedCard`、辅助函数 `buildQuestionActions` |
| 修改 | `test/feishu-cards.test.js` | 新增 buildQuestionCard/buildQuestionRepliedCard 测试 |

## 复杂度

high，原因：4 种 inputMode 对应不同飞书组件（button/multi_select_static/input），需处理 options 缺失、required 校验、未知 inputMode 降级、form_value key 约定；飞书卡片组件类型未在代码库出现先例，需验证协议可用性。

## 实现步骤

- [ ] 步骤 1：在 `cards.js` 新增 `buildQuestionCard(questionEvent, sessionId, routeKey)`：
  - 从 `questionEvent.data` 提取 `id`（questionId）、`title`、`metadata.inputMode`、`metadata.description`、`metadata.options`、`metadata.required`。
  - header template 用 `'blue'`（与 permission 的 `'red'` 区分），title 用 `data.title || '交互式问题'`。
  - elements 第一项为 `div` 展示标题和 description（用 `escapeLarkMd` 转义）。
  - 根据 inputMode 构造 actions：
    - `confirm`：2 个 button（允许/拒绝），value 用 `buildButtonValue('cmd:/answer <id> allow|deny', sessionId, routeKey)`，type 分别 `primary`/`danger`。
    - `single_select`：N 个 button（每个 option 一个），value 用 `buildButtonValue('cmd:/answer <id> <value>', sessionId, routeKey)`，label 用 `option.label || option.value`。
    - `multi_select`：1 个 `multi_select_static` 组件（options 映射为 `{ text: { tag: 'plain_text', content: label }, value: option.value }`，name 为 `'question_answer'`）+ 1 个提交 button，button value 用 `buildButtonValue('cmd:/answer <id> --form', sessionId, routeKey)`。
    - `text`：1 个 `input` 组件（name 为 `'question_answer'`，placeholder 用 description 或 `'请输入'`）+ 1 个提交 button，button value 用 `buildButtonValue('cmd:/answer <id> --form', sessionId, routeKey)`。
    - 未知 inputMode：降级为 `confirm`，记录 warn（通过 logger，但 cards.js 无 logger，降级只影响渲染，warn 在 PermissionHandler 层记录）。
  - 缺少 options（select 类型）时：渲染错误状态卡片（`div` 显示 `'选项缺失，无法渲染问题卡片'`），不生成 actions。
- [ ] 步骤 2：新增 `buildQuestionRepliedCard(questionId, answer)`：
  - header template 用 `'green'`，title `'问题已回复'`。
  - elements 为 `div` 展示 `'已回答: ' + formatAnswer(answer)`，其中 `formatAnswer` 对 string[] 用 `answer.join(', ')`，string 直接返回。
  - 不含 actions（已处理状态）。
  - 注：不传 inputMode 参数 — replied 卡片只展示回答摘要，inputMode 在已处理状态无展示用途。
- [ ] 步骤 3：在 `module.exports` 中导出 `buildQuestionCard` 和 `buildQuestionRepliedCard`。
- [ ] 步骤 4：用飞书卡片预览工具或飞书开放平台文档验证 `multi_select_static` 和 `input` 组件的 `form_value` 回传结构（spec §4.3 约定 `question_answer` key），确认组件可用性。若协议有差异，调整 form_value key 和组件结构，并在测试中固定。
