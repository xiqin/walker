# Task 8: 端到端集成测试与向后兼容验证

## 描述

新建端到端集成测试，验证从 TUI bridge 事件上报 → 飞书卡片渲染 → 用户点击/提交 → /answer 命令 → driver 回复 → 卡片更新的全链路，覆盖 4 种 inputMode、幂等保护、向后兼容和异常场景。

## 依赖

- 前置 task：T1-T7 全部（含合并后的 Dispatcher + PermissionHandler）
- 外部依赖：无

| 操作 | 文件路径 | 说明 |

## 复杂度

high，原因：需 mock 飞书 API、TUI bridge、driver registry、session service 等多个组件，验证全链路状态流转和卡片更新；覆盖 4 种 inputMode 和异常场景。

## 实现步骤

- [ ] 步骤 1：创建 `test/integration-feishu-question.test.js`，搭建测试 fixture：
  - mock FeishuApi（replyCard/patchCard/replyText）
  - mock SessionService（getCurrent/getRouteForSession）
  - mock DriverRegistry（get 返回 mock OpencodeDriver）
  - 真实 MessageDispatcher + PermissionHandler + cards.js
- [ ] 步骤 2：测试用例组 — 全链路成功路径：
  - `confirm: TUI bridge 上报 question 事件 → buildQuestionCard 渲染 → 点击允许 → /answer <id> allow → driver.replyPermission → patch 卡片`
  - `single_select: 上报事件 → 渲染 N 按钮 → 点击选项 → /answer <id> <value> → 回传 → patch`
  - `multi_select: 上报事件 → 渲染 multi_select_static → 提交 formValue → /answer <id> --form → 回传 string[] → patch`
  - `text: 上报事件 → 渲染 input → 提交文本 → /answer <id> --form → 回传 string → patch`
- [ ] 步骤 3：测试用例组 — 幂等与状态：
  - `重复点击 replied 状态返回已处理提示`
  - `submitting 状态返回处理中提示`
  - `driver 回传失败回滚为 pending 允许重试`
- [ ] 步骤 4：测试用例组 — 向后兼容：
  - `传统 permission 事件（无 data.type=question）用 buildPermissionCard + /permit 不受影响`
  - `现有 /permit allow|deny 流程不受 answer 命令影响`
  - `传统 permission 的 permission_replied 事件走 handleReplied（buildPermissionRepliedCard）不走 handleQuestionReplied`
  - `question 的 permission_replied 事件走 handleQuestionReplied（buildQuestionRepliedCard）不走 handleReplied`
- [ ] 步骤 5：测试用例组 — 异常场景：
  - `未知 inputMode 降级为 confirm`
  - `select 缺少 options 渲染错误状态`
  - `text required 空答案拒绝`
  - `multi_select required 空数组拒绝`
  - `TUI bridge runtime 不在线 回复失败并回滚 pending`
  - `TUI bridge replyQuestion lease 超时 回滚 pending`
  - `无 session 绑定`
  - `driver 不支持 replyPermission`
- [ ] 步骤 6：测试用例组 — patch 卡片策略（spec §3.2.2 兜底关系）：
  - `HTTP transport _cmdAnswer 成功后直接 patch 卡片`
  - `TUI bridge transport _cmdAnswer 成功后不 patch，等 permission_replied 事件驱动 handleQuestionReplied patch`
  - `TUI bridge transport runtime 未上报 permission_replied 时卡片不更新（兜底语义）`
- [ ] 步骤 6：测试用例组 — TUI bridge 回复 delivery：
  - `TUI bridge transport replyQuestion 投递 question_reply delivery`
  - `runtime poll 拉取 delivery`
  - `accepted → heartbeat → final 全流程`
  - `final error 回滚状态`
