# Task 1: 卡片结构与描述渲染

## 描述

实现 `buildNativeQuestionCard()` 的混合卡片输出：单选保持 legacy v1 普通按钮，多选输出完整 Card JSON 2.0 表单，并在正文中展示选项说明。

## 依赖

- 前置 task：无
- 外部依赖：飞书 Card JSON 1.0/2.0 字段约束

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/platform/feishu/cards.js` | 增加选项说明 Markdown 构建；按题型输出单选 v1 卡片或多选 v2 表单卡片；无预设题只提示本地 TUI 输入。 |
| 修改 | `test/feishu-cards.test.js` | 更新卡片结构测试，锁定单选 v1、多选 v2、描述展示和禁止混用字段。 |

## 复杂度

medium，原因：同一构建函数需要稳定维护 v1 与 v2 两套卡片结构，并避免再次引入飞书 400 的混用字段。
