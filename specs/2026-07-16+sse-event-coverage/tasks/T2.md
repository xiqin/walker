# Task 2: mapSSEEvent 全量 SSE 事件映射

## 描述

在 opencode-sse-adapter.js 的 mapSSEEvent 函数中新增所有未覆盖的 SSE 事件类型映射分支，将 OpenCode 1.17.20 全部 31 种事件映射到 AgentEvent 体系或静默丢弃。

## 依赖

- 前置 task：T1（AgentEvent 新增 TYPE 常量）
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/drivers/opencode-sse-adapter.js` | mapSSEEvent 新增 20+ 事件映射分支 |
| 创建 | `test/opencode-sse-adapter.test.js` | 新增适配层单元测试 |

## 复杂度

high，原因：20+ 种事件映射分支，需处理多种 part.type 分级、字段提取差异、边界场景

## 实现步骤

- [ ] 步骤 1：新增 `permission.updated` 映射 → TYPE_PERMISSION，提取 id/type/title/metadata/sessionID/messageID/callID
- [ ] 步骤 2：新增 `permission.replied` 映射 → TYPE_PERMISSION_REPLIED，提取 permissionId/response
- [ ] 步骤 3：新增待办更新事件（SSE 类型名 `待办-dot-updated`）映射 → `TYPE_待办`（AgentEvent 待办类型），提取 todos 数组
- [ ] 步骤 4：新增 `session.compacted` 映射 → TYPE_COMPACTED，提取 sessionID
- [ ] 步骤 5：新增 `file.edited` 映射 → TYPE_FILE_EDITED，提取 path/action/linesAdded/linesRemoved
- [ ] 步骤 6：新增 `session.diff` 映射 → TYPE_SESSION_DIFF，提取 diff/filesCount/linesAdded/linesRemoved
- [ ] 步骤 7：扩展 `message.part.updated` 分支，新增未识别 part.type 处理：
  - step-start/step-finish → TYPE_STEP，提取 partType/stepId
  - file → TYPE_FILE_EDITED，提取 path
  - patch → TYPE_FILE_EDITED，提取 path/action
  - snapshot/agent/retry/compaction/subtask → return null（静默丢弃，debug 日志）
- [ ] 步骤 8：新增 `message.removed` / `message.part.removed` 映射 → TYPE_MESSAGE_REMOVED，提取 messageId/partId
- [ ] 步骤 9：新增 `command.executed` 映射 → TYPE_COMMAND_EXECUTED，提取 command/exitCode
- [ ] 步骤 10：新增 `session.created` / `session.updated` / `session.deleted` 映射 → TYPE_SESSION_LIFECYCLE，提取 action/session
- [ ] 步骤 11：新增 `server.connected` 映射 → TYPE_SERVER_CONNECTED
- [ ] 步骤 12：将 `installation.*` / `lsp.*` / `vcs.branch.updated` / `file.watcher.updated` / `tui.*` / `pty.*` / `server.instance.disposed` 显式 return null 并记录 debug 日志
- [ ] 步骤 13：保留最后兜底 return null（未来新增未知事件）
