# Task 1: AgentEvent 类型体系扩展

## 描述

在 agent-driver.js 中新增 11 种 AgentEvent TYPE 常量和对应 DATA_SCHEMAS 声明，不影响现有 6 种类型。

## 依赖

- 前置 task：无
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/drivers/agent-driver.js` | 新增 11 种 EVENT_TYPE 常量、DATA_SCHEMAS 声明、TYPE 静态属性 |

## 复杂度

medium，原因：纯类型声明扩展，无逻辑变更，但需保证与现有 6 种类型共存不冲突

## 实现步骤

- [ ] 步骤 1：在现有 6 个 EVENT_TYPE_* 常量后新增 11 个常量：
  - `EVENT_TYPE_PERMISSION = 'permission'`
  - `EVENT_TYPE_PERMISSION_REPLIED = 'permission_replied'`
  - `EVENT_TYPE_TODO` 常量，值为 `'待办事件'` 字符串（spec 3.3 节定义的 type 值）
  - `EVENT_TYPE_COMPACTED = 'compacted'`
  - `EVENT_TYPE_FILE_EDITED = 'file_edited'`
  - `EVENT_TYPE_SESSION_DIFF = 'session_diff'`
  - `EVENT_TYPE_STEP = 'step'`
  - `EVENT_TYPE_MESSAGE_REMOVED = 'message_removed'`
  - `EVENT_TYPE_COMMAND_EXECUTED = 'command_executed'`
  - `EVENT_TYPE_SESSION_LIFECYCLE = 'session_lifecycle'`
  - `EVENT_TYPE_SERVER_CONNECTED = 'server_connected'`
- [ ] 步骤 2：在 DATA_SCHEMAS 中为每种新类型添加字段声明（按 spec 3.3 节）：
  - permission: `{ id: 'string', type: 'string', title: 'string', metadata: 'object?', sessionID: 'string', messageID: 'string', callID: 'string?' }`
  - permission_replied: `{ permissionId: 'string', response: 'string' }`
  - 待办事件: `{ todos: 'object[]' }`（DATA_SCHEMAS 的 key 为待办事件 type 值）
  - compacted: `{ sessionID: 'string' }`
  - file_edited: `{ path: 'string', action: 'string', linesAdded: 'number?', linesRemoved: 'number?' }`
  - session_diff: `{ diff: 'string', filesCount: 'number', linesAdded: 'number', linesRemoved: 'number' }`
  - step: `{ partType: 'string', stepId: 'string?' }`
  - message_removed: `{ messageId: 'string', partId: 'string?' }`
  - command_executed: `{ command: 'string', exitCode: 'number' }`
  - session_lifecycle: `{ action: 'string', session: 'object' }`
  - server_connected: `{}`
- [ ] 步骤 3：为每种新类型添加 `AgentEvent.TYPE_*` 静态属性赋值
