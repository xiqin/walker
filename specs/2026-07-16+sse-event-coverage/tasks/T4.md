# Task 4: 进度卡片 + 权限卡片渲染扩展

## 描述

扩展 progress-card.js 的 formatAgentEvent switch 新增 11 种 case，在 cards.js 新增 buildPermissionCard 构建权限确认交互卡片（独立于进度卡片）。

## 依赖

- 前置 task：T1（AgentEvent 新增 TYPE 常量）
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/platform/feishu/progress-card.js` | formatAgentEvent 新增 11 种 case |
| 修改 | `src/platform/feishu/cards.js` | 新增 buildPermissionCard 函数 |
| 修改 | `test/progress-card.test.js` | 新增 formatAgentEvent 扩展测试 |

## 复杂度

medium，原因：11 种 case 逻辑简单但数量多，权限卡片构建涉及飞书卡片 JSON 结构

## 实现步骤

- [ ] 步骤 1：在 formatAgentEvent switch 中新增 case（参照 spec 4.2 节展示规则）：
  - `permission` → 返回 '' （权限卡片独立处理，不进进度卡片 entries）
  - `permission_replied` → 返回 ''（同上）
  - 待办事件 → `📋 待办: <done>/<total> 完成`（从 data.todos 计算完成数）
  - `compacted` → `🗜️ 上下文已压缩`
  - `file_edited` → `📝 已编辑 <path>` 或累计计数
  - `session_diff` → `📊 diff: <filesCount> 文件, +<linesAdded> -<linesRemoved>`
  - `step` → partType 为 step-start 返回 `▶ 步骤: <stepId>`, step-finish 返回 `✅ 步骤: <stepId> 完成`
  - `message_removed` → 返回 ''（不展示）
  - `command_executed` → `⬇ 命令: <command> (exit <exitCode>)`
  - `session_lifecycle` → 返回 ''（不展示）
  - `server_connected` → 返回 ''（不展示）
- [ ] 步骤 2：在 cards.js 新增 `buildPermissionCard(permissionEvent, sessionId, routeKey)` 函数：
  - header 红色模板，title "权限确认请求"
  - body 显示 `data.title` 和 `data.metadata` 内容
  - 按钮区两个按钮："允许"（type: primary）和"拒绝"（type: danger）
  - 按钮 value 使用 `buildButtonValue('cmd:/permit <permissionId> allow', sessionId, routeKey)` 和 deny 版本
  - 缺少 title 时显示"未知权限请求"（spec 6 节边界场景）
- [ ] 步骤 3：在 cards.js 新增 `buildPermissionRepliedCard(permissionId, response)` 函数：
  - header 灰色模板，title "权限已处理"
  - body 显示"已<允许/拒绝>权限请求 <permissionId>"
- [ ] 步骤 4：导出新增函数
