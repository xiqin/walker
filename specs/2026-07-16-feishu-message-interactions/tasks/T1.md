# Task 1: 统一模型视图与 OpenCode Recent 映射

## 描述

让 agent driver 暴露统一模型目录结构，并让 OpenCode driver 将接口返回转换为该结构，供后续飞书卡片和 dispatcher 使用。

## 依赖

- 前置 task：无
- 外部依赖：本地 OpenCode `/api/model` 响应结构，测试中通过 fake httpClient 模拟

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/drivers/agent-driver.js` | 明确或实现 `listModels()` 默认契约，不支持时抛出可诊断错误。 |
| 修改 | `src/drivers/opencode-driver.js` | 输出统一模型视图字段：`id`、`name`、`provider`、`status`、`enabled`、`source`、`groups`、`lastUsedAt`。 |
| 修改 | `test/opencode-driver.test.js` | 增加统一模型视图和 Recent 映射测试。 |

## 复杂度

medium，原因：字段映射需要兼容 OpenCode 可能返回的多种命名，同时要避免飞书层依赖 OpenCode 原始字段。

## 实现步骤

- [ ] 步骤 1：在 `AgentDriver` 中声明 `listModels()` 默认不支持行为，错误信息包含 driver 不支持模型目录的语义。
- [ ] 步骤 2：在 `OpencodeDriver.listModels()` 中保留现有 id/name/provider/status/enabled 兼容映射，并追加 `source: 'opencode'`。
- [ ] 步骤 3：将 OpenCode Recent 元数据映射为统一字段；识别 `groups` 包含 `recent`、`group/category` 等于 `recent`、`recent === true` 或存在 `lastUsedAt` 的情况。
- [ ] 步骤 4：保持 `status === 'deprecated'` 或 `enabled === false` 后续可被上层过滤；driver 层继续过滤无 id 或 disabled 的无效模型，避免破坏现有 `/model` 解析。
- [ ] 步骤 5：补充测试确认普通模型、Recent 模型、lastUsedAt、source 字段均被保留。
