# Task 2: 飞书模型、帮助卡片与 API 挂载

## 描述

在飞书平台层新增模型选择卡片和帮助卡片，导出命令元数据，并把卡片发送能力挂载到飞书 API 目标。

## 依赖

- 前置 task：T1
- 外部依赖：飞书交互卡片 JSON 结构约定

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/platform/feishu/cards.js` | 新增 `renderModelListCard(models, options)`、`renderHelpCard(commands, options)` 并导出。 |
| 修改 | `src/platform/feishu/commands.js` | 导出命令元数据供帮助卡片使用，同时保持 `formatHelp()` 和 `parseCommand()` 行为。 |
| 修改 | `src/app/bootstrap.js` | 导入新卡片渲染函数，挂载 `sendModelList`、`sendHelpCard`。 |
| 修改 | `test/feishu-cards.test.js` | 增加模型卡片和帮助卡片渲染测试。 |

## 复杂度

medium，原因：需要控制按钮数量、Recent 分组、provider 分组、routeKey 与飞书卡片格式。

## 实现步骤

- [ ] 步骤 1：新增常量 `MAX_MODEL_CARD_ITEMS = 20` 与 `MAX_RECENT_MODEL_ITEMS = 5`，限制模型按钮数量。
- [ ] 步骤 2：实现模型过滤：跳过空模型、`status === 'deprecated'`、`enabled === false`。
- [ ] 步骤 3：实现 Recent 区块：只根据统一模型视图中的 `groups` 包含 `recent` 或 `lastUsedAt` 排序展示，最多 5 个。
- [ ] 步骤 4：实现 provider 分组按钮，按钮 value 为 `buildCommandValue('cmd:/model <provider>/<id>', routeKey)` 或无 provider 时 `cmd:/model <id>`。
- [ ] 步骤 5：超过 20 个可展示模型时，在卡片顶部提示使用 `/model <provider>/<model_id>` 精确切换。
- [ ] 步骤 6：在 `commands.js` 中导出可迭代命令元数据，避免帮助卡片重复硬编码命令清单。
- [ ] 步骤 7：实现帮助卡片，输入命令元数据后展示 usage/desc 并为每条命令生成 `cmd:/<name>` 按钮。
- [ ] 步骤 8：在 `bootstrap.js` 中挂载 `sendModelList(replyCtx, models, options)` 和 `sendHelpCard(replyCtx, commands, options)`。
