# Task 3: 飞书 `/clear` 命令与调度

## 描述

将 `/clear` 注册到飞书命令表，并在 dispatcher 中通过锁外快速检查与 route lock 内复检，安全执行当前 TUI session 的 clear 操作。

## 依赖

- 前置 task：T1、T2
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/platform/feishu/commands.js` | 增加 clear 命令定义和帮助说明 |
| 修改 | `src/dispatch/message-dispatcher.js` | 增加 clear handler、业务校验、成功回复和 route 串行化 |
| 修改 | `test/feishu-commands.test.js` | 验证解析、命令清单和帮助文本 |
| 修改 | `test/message-dispatcher.test.js` | 验证成功及所有不支持场景，不调用 createSession/openTerminal |

## 复杂度

medium，原因：需要与现有 prompt queue、turn state、session 状态和 route lock 的语义保持一致。

## 实现步骤

- [ ] 在 `COMMANDS` 增加无参数 `/clear`，描述为在当前 TUI 新建空上下文并保留旧会话。
- [ ] 在 `handleCommand()` 中先调用轻量 preflight：读取当前 route 焦点；无绑定、session 为 `running`、存在 turn state 或存在未完成 prompt queue 时立即回复，不能先排队等待 route lock。
- [ ] preflight 通过后，让 clear 与 `/new`、`/attach` 一样进入 `_enqueueRouteLock()` 和 `_withRouteTouch()`。
- [ ] 新增 `_cmdClear(cmd)`：获得锁后重新读取当前 route 焦点，并重复无绑定、session 身份、transport、running、turn 和 prompt queue 校验；不得使用 preflight 缓存的 session 对象。
- [ ] 校验当前 session 为 `agent=opencode` 且 `agentRef.transport='tui-bridge'`，driver 实现 `clearSession()`；禁止回退到 `createSession()`。
- [ ] 任一空闲检查失败时提示先执行 `/cancel`，不等待、不自动取消、不在 prompt 结束后自动执行 clear。
- [ ] 调用 `driver.clearSession(current.agentRef)`；成功后确认返回的新 Walker session，继承逻辑由 bridge 保证。
- [ ] 成功回复包含旧、新 Walker session ID 和当前 TUI 窗口保持不变；错误沿用 `handleCommand()` 的错误卡片路径。
- [ ] 更新命令注释及测试桩，明确断言 `/new`、driver `createSession()` 和 runtime `openTerminal()` 未被调用。
