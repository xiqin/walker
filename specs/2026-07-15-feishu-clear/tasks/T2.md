# Task 2: TUI plugin 执行 clear 控制

## 描述

升级自动生成的 OpenCode TUI plugin，使其区分 prompt 与 clear delivery，并在当前 runtime 内创建顶层 session、导航、发送关联注册和控制结果；失败时最佳努力回滚到旧 session。

## 依赖

- 前置 task：T1
- 外部依赖：OpenCode TUI SDK `api.client.session.create()`、`api.route.navigate()`

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/opencode-hook/plugin-template.js` | 实现 clear 控制处理、session ID 兼容提取和结果上报 |
| 修改 | `test/opencode-hook-installer.test.js` | 执行生成插件并验证创建、导航、注册和错误上报 |

## 复杂度

medium，原因：生成代码需兼容 OpenCode SDK 多种返回形态、抑制 clear 自身的通用自动注册，并协调关联注册、控制上报和失败回滚。

## 实现步骤

- [ ] 提升 TUI bridge plugin 版本，确保已安装旧版本在 Walker 启动时自动更新。
- [ ] 将 delivery 分派为 `type === 'clear'` 和 prompt；缺少 `type` 时继续按 prompt 执行。
- [ ] clear handler 调用 `api.client.session.create({ title })` 创建独立顶层 session，不传 parent，不执行 fork、summarize 或 delete。
- [ ] 从 `result.data.id`、`result.id`、`result.sessionID`、`result.sessionId` 提取新 session ID，缺失时按错误上报。
- [ ] clear 调用 create 前即设置 active clear 状态并缓冲 `session.created` 和 `tui.session.select`；取得返回 ID 后只抑制与新 ID 对应的普通自动注册，确保事件早于 create Promise 返回时也不提前改变 `activeSessionId` 或注册。
- [ ] 创建成功后调用 `api.route.navigate('session', { sessionID: newId })`；导航成功后分别发送带 `controlDeliveryId` 的 register 与 `{ control: { type: 'clear', newSessionId } }`，不得依赖二者顺序。
- [ ] 关联 register 与 control success 都成功后，更新插件的 `activeSessionId` 和普通注册缓存，再处理仍然有效的非 clear 缓冲事件；避免把 clear 加入 `activeDeliveries` prompt 状态。
- [ ] 创建、导航、关联注册或控制上报失败时发送现有 `error` 结构；若已经导航，则最佳努力调用 `api.route.navigate('session', { sessionID: oldSessionId })` 回滚。回滚成功后恢复旧 `activeSessionId`，回滚失败时错误信息追加需手工切回的提示；最后清理 active clear 并安全处理非 clear 缓冲事件，确保 poll loop 可继续工作。
