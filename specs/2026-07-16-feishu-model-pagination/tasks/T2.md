# Task 2: 分页命令与原卡片更新集成

## 描述

将 `/model --page N` 识别为模型目录分页请求，绕过普通命令去重，并通过触发消息 ID 更新原卡片。

## 依赖

- 前置 task：T1
- 外部依赖：飞书卡片回调提供 `messageId`，`FeishuApi.patchCard(messageId, card)` 已存在

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/dispatch/message-dispatcher.js` | 解析分页命令、跳过去重、传递页码和原卡片消息 ID。 |
| 修改 | `src/app/bootstrap.js` | `sendModelList` 根据 `updateMessageId` 选择 `patchCard` 或 `replyCard`。 |
| 修改 | `test/message-dispatcher.test.js` | 覆盖分页命令、往返、fallback 和模型切换兼容。 |
| 修改 | `test/bootstrap.test.js` | 覆盖模型卡片新建与原地更新分支。 |

## 复杂度

high，原因：同一 `/model` 命令同时承载列表、分页和模型切换三种语义，并涉及去重和飞书回复方式的协调。

## 实现步骤

- [ ] 步骤 1：补充 `/model --page 2` 的 dispatcher 失败测试，确认现有代码会误走模型切换。
- [ ] 步骤 2：在普通命令去重前严格识别模型分页动作，仅分页动作跳过去重。
- [ ] 步骤 3：让 `_cmdModel` 复用模型目录读取路径并传递 `page`、`currentModel`、`routeKey` 和 `updateMessageId`。
- [ ] 步骤 4：保证非法页码不会更新 session.model，由卡片层执行安全归一化。
- [ ] 步骤 5：修改 bootstrap 的 `sendModelList`，首次打开使用 `replyCard`，分页动作使用 `patchCard`。
- [ ] 步骤 6：补充 patch 失败纯文本 fallback、页面往返不去重和普通模型按钮仍去重的测试。
- [ ] 步骤 7：回归直接模型切换和 routeKey 行为并运行全量检查。
