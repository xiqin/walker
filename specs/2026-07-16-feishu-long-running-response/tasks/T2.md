# Task 2: HTTP/SSE 超时原语

## 描述

把底层 SSE 固定总时长改为空闲超时，并保证 HTTP 请求与 SSE 都支持取消信号和稳定错误码。

## 依赖

- 前置 task：无
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
| ---- | -------- | ---- |
| 修改 | `src/core/http-helper.js` | 为 SSE 实现 chunk 续期的 idle timer，为 HTTP/SSE timeout 与 abort 设置稳定错误码。 |
| 修改 | `src/drivers/opencode-http-client.js` | 保持 `timeoutMs` 与 `signal` 选项显式透传。 |
| 修改 | `test/http-helper.test.js` | 覆盖 SSE 空闲续期、空闲失败、HTTP timeout 和 abort。 |
| 创建 | `test/opencode-http-client.test.js` | 覆盖默认 client 的 timeout/signal 透传契约。 |

## 复杂度

medium，原因：涉及异步资源清理、竞态与定时器重置，必须保证 Promise 只 settle 一次。

## 实现步骤

- [ ] 将 `sseConnect()` 的整流 `timeoutMs` 语义替换为 `idleTimeoutMs`，仅在成功建立 SSE response 后启动空闲 timer。
- [ ] 每次收到任意 `data` chunk 时重置 idle timer，包括注释 keepalive 和尚未组成完整事件的 chunk。
- [ ] timeout、abort、response end、request error 和正常 terminal event 共用幂等 cleanup，清理 timer、listener、response 与 request。
- [ ] 为 SSE idle timeout 设置 `error.code='SSE_IDLE_TIMEOUT'`，为 abort 设置 `error.code='ABORT_ERR'`；HTTP request timeout 保留独立 `timeoutMs` 并提供可识别 code。
- [ ] 确认 `DefaultHttpClient.request()` 不覆盖调用方传入的 `timeoutMs: 0` 或 `signal`，`DefaultSSEClient.connect()` 原样传递 `idleTimeoutMs`。
