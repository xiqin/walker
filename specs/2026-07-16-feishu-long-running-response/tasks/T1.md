# Task 1: 配置解析与依赖注入

## 描述

新增分层 transport 超时和 TUI 租约配置，保留旧配置兼容，并通过 bootstrap 与 plugin installer 显式注入各组件。

## 依赖

- 前置 task：无
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
| ---- | -------- | ---- |
| 修改 | `src/config/env.js` | 增加允许 `0` 的非负整数解析，读取新配置并实现旧配置 fallback。 |
| 修改 | `src/app/bootstrap.js` | 使用 `??` 或显式 `undefined` 判断，把新配置传给 Driver、Bridge 和 plugin installer。 |

| 修改 | `.env.example` | 增加新配置示例及安全默认值。 |

## 复杂度

medium，原因：配置项较多且需要兼容旧部署，但实现集中在既有配置与装配边界。

## 实现步骤

- [ ] 在 `loadEnvConfig()` 内增加非负整数解析，允许 open/request/idle/lease timeout 显式为 `0`；heartbeat interval 继续要求正整数。
- [ ] 定义 `opencodePromptRequestTimeoutMs=30000`、`opencodeSseIdleTimeoutMs=300000`、`opencodeTuiLeaseTimeoutMs=90000`、`opencodeTuiHeartbeatIntervalMs=30000`，并让 `opencodeSseOpenTimeoutMs` 接受 `0`。
- [ ] 当 `OPENCODE_SSE_IDLE_TIMEOUT_MS` 未提供而 `OPENCODE_PROMPT_TIMEOUT_MS` 已提供时，仅将旧值作为 idle timeout；两者都未提供时使用 `300000`。
- [ ] 对 `leaseTimeoutMs > 0` 且 `heartbeatIntervalMs >= leaseTimeoutMs` 的配置抛出包含两个变量名的明确错误，避免静默产生必然掉租的配置。
- [ ] bootstrap 按接口契约构造 Driver 与 Bridge，不使用会吞掉 `0` 的 `||`；plugin installer 接收 heartbeat interval。
- [ ] 更新 README 和 `.env.example`，明确 `WALKER_MAX_TURN_TIME_MINS` 是唯一整轮硬截止，旧 `OPENCODE_PROMPT_TIMEOUT_MS` 已废弃。
