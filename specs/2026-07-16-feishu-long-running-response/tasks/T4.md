# Task 4: TUI Bridge v3 租约与 tombstone

## 描述

将 prompt delivery 从固定 timeout 改为 v3 租约状态机，并用有界 tombstone 对重复或迟到 final 做幂等补投或抑制。

## 依赖

- 前置 task：T1
- 外部依赖：内置 plugin 按 Task 5 的 v3 上报契约工作

## 涉及文件

| 操作 | 文件路径 | 说明 |
| ---- | -------- | ---- |
| 修改 | `src/opencode-tui-bridge/bridge.js` | 实现 queued/leased/completed/cancelled/lease_lost 状态、租约续期、取消与 tombstone。 |
| 修改 | `src/opencode-tui-bridge/routes.js` | 保持 events endpoint 将 `deliveryState` 原样传给 Bridge，并返回幂等结果。 |
| 修改 | `test/opencode-tui-bridge.test.js` | 覆盖 v3、v2、租约、迟到 final、取消与 runtime dispose。 |

## 复杂度

high，原因：delivery 生命周期包含 timer、runtime/session 绑定、watcher 补投和重复 final 竞态。

## 实现步骤

- [ ] Bridge 构造函数接收 `leaseTimeoutMs` 与 tombstone 容量/过期安全默认值，使用 `??` 保留 `0`；移除 prompt 固定总时长。
- [ ] `prompt(sessionRef, text, { signal })` 创建 queued pending，保存 delivery/runtime/session/state/resolve/reject/timer/cancel reason；signal abort 时进入 cancelled。
- [ ] `accepted` 仅允许匹配的 v3 queued delivery 转为 leased 并启动租约；`heartbeat` 仅对匹配的 leased delivery续租，不 resolve、不投递 watcher。
- [ ] `final` 清理租约并 resolve/reject prompt；未提供 `deliveryState` 的 v2 上报按 final 处理，且 v2 不施加租约或固定总时长。
- [ ] 租约到期将 delivery 转为 `transport_lost` tombstone，reject 原 prompt 并使用 `TUI_RUNTIME_DISCONNECTED`；runtime dispose/close 采用同类可恢复 tombstone，用户取消和 turn deadline 使用抑制型 tombstone。
- [ ] transport_lost 的迟到 final 通过对应 `watchers` 转交事件且至多一次；cancel/deadline 的迟到 final 返回幂等成功但不投递；completed 的重复 final 幂等忽略。
- [ ] runtimeId/sessionId/deliveryId 不匹配时仍抛明确错误，不能将结果路由到其他 session。
- [ ] tombstone 使用容量上限与过期清理；所有 terminal 分支清理 timer 和 abort listener。
- [ ] 保持 `/clear` control delivery 现有语义，仅复用安全的 timer cleanup，不把 prompt v3 状态混入 clear 协议。
