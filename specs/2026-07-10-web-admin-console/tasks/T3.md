# Task 3: Session、Route、Agent 与 Runtime API

## 描述

实现管理端核心业务 API，复用 `SessionService`、`DriverRegistry` 和 runtime 配置，覆盖总览、session、route、agent、runtime、timeline 和 prompt。

## 依赖

- 前置 task：T1、T2
- 外部依赖：现有 Walker service 和 driver 实例

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/admin/core-routes.js` | 导出核心 API route 列表，供最终集成组装 |
| 创建 | `src/admin/session-admin.js` | session 列表、详情、创建、停止、删除、prompt、timeline 服务函数 |
| 创建 | `src/admin/route-admin.js` | route 列表、绑定、解绑、悬空诊断、清理服务函数 |
| 创建 | `src/admin/agent-runtime-admin.js` | driver 摘要、OpenCode 检测、自启、runtime 检测服务函数 |
| 创建 | `test/admin-core-api.test.js` | 使用 stub service/driver 覆盖核心 API 行为 |

## 复杂度

high，原因：需要协调持久化 session、driver 调用、route 健康状态和 prompt 时间线，错误路径较多。

## 实现步骤

- [ ] 实现 `GET /api/admin/overview`，返回进程、数据目录、session/route 统计、driver/runtime/feishu 摘要和最近错误。
- [ ] 实现 sessions API：列表、详情、创建、停止、删除、prompt、timeline；停止和删除优先调用 driver，失败时返回 warning 并更新 Walker 状态。
- [ ] 实现 routes API：列出健康状态、绑定到已有 session、按 encoded routeKey 解绑、确认后清理悬空绑定。
- [ ] 实现 agents API：列出 `opencode/claude/codex` 状态，stub driver 返回不可用原因，OpenCode 支持 check 与 ensure-ready。
- [ ] 实现 runtime API：返回 Windows/WSL 配置、cwd 存在性、WSL distro 和 WSL IP 探测摘要。
- [ ] 将 session 状态变更、route 绑定、prompt、driver 错误写入 event store，供 events、metrics 和 timeline 读取。
