# Task 2: Admin HTTP 服务骨架与鉴权

## 描述

实现可测试的 Admin HTTP 服务，包含启动关闭、JSON 响应格式、Bearer token/cookie 鉴权、静态资源服务和 SPA fallback。

## 依赖

- 前置 task：T1
- 外部依赖：Node.js 内置 `http`、`url`、`fs`、`path` 模块

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/admin/server.js` | `AdminServer` 生命周期、请求解析、路由分发、静态资源托管 |
| 创建 | `src/admin/router.js` | 轻量 method/path 路由匹配，支持 `:id` 参数 |
| 创建 | `src/admin/response.js` | 统一 `{ ok, data }` 与 `{ ok, error }` JSON 响应 |
| 创建 | `src/admin/auth.js` | token 检查、登录接口辅助、写操作鉴权中间逻辑 |
| 创建 | `src/admin/static.js` | 静态文件 MIME、路径限制、SPA fallback 与 404 处理 |
| 创建 | `test/admin-server.test.js` | 覆盖启动、关闭、鉴权、JSON 格式、静态文件、安全路径 |

## 复杂度

high，原因：需要自行实现 HTTP 边界、鉴权和路径安全，且要保证管理端异常不影响主流程。

## 实现步骤

- [ ] 设计 `AdminServer` 构造参数：`config`、`routes`、`publicDir`、`eventStore`、`now`、`serverFactory`。
- [ ] 实现 `start()` 与 `stop()`，在 `adminEnabled=false` 时不监听端口并返回禁用状态。
- [ ] 实现 `/api/admin/auth/status` 和 `/api/admin/auth/login`，支持 `Authorization: Bearer <token>` 与安全 cookie。
- [ ] 对写操作要求 token；未配置 token 时仍只按本机绑定设计运行，响应不返回敏感 secret。
- [ ] 实现静态文件响应、`/` 返回 `index.html`、SPA 子路径 fallback、API 路径不落入静态 fallback。
- [ ] 所有 API 错误统一为 spec 中的 JSON 格式，并记录 error 事件。
