# Task 5: 调试工具、卡片预览、指标与服务控制 API

## 描述

实现网页管理端的调试工具类 API，包括飞书命令模拟、卡片预览、指标读取和受控停止 Walker。

## 依赖

- 前置 task：T1、T2、T3
- 外部依赖：现有飞书 command 与 card 渲染模块

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/admin/tools-routes.js` | tools、cards、metrics、service stop route 导出 |
| 创建 | `src/admin/command-simulator.js` | `/new`、`/list`、`/use` 等命令解析和动作摘要 |
| 创建 | `src/admin/card-preview.js` | 卡片类型列表、示例数据、JSON 与简化视觉预览数据 |
| 创建 | `src/admin/service-control.js` | 二次确认后调用 app stop 并延迟退出进程 |
| 创建 | `test/admin-tools.test.js` | 覆盖命令模拟、卡片预览、指标和服务停止确认 |

## 复杂度

medium，原因：主要复用现有解析和渲染模块，但服务停止必须严格要求确认并便于测试注入。

## 实现步骤

- [ ] 实现命令模拟器，返回 `parseCommand` 结果、模拟 routeKey、dispatcher 动作摘要和 dry-run 标记。
- [ ] 实现卡片类型列表和渲染接口，覆盖未绑定引导、session 列表、可纳入 session、错误卡片和进度卡片。
- [ ] 实现 metrics route，读取 T1 event store 的计数和 60 分钟趋势。
- [ ] 实现服务停止 route，要求 `{ confirm: true }`，响应后调用注入的 `stopApp` 并通过注入的 `exitProcess` 延迟退出。
- [ ] 将 admin action 和错误写入事件 store。
