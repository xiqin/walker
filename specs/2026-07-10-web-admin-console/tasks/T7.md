# Task 7: 应用集成、脚本收口与全量验证

## 描述

把所有管理端模块组装进 Walker 应用生命周期，更新检查脚本，补充端到端式 HTTP 集成测试并执行全量验证。

## 依赖

- 前置 task：T1、T2、T3、T4、T5、T6
- 外部依赖：本地端口监听能力

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/admin/index.js` | 组装 AdminServer、全部 route、publicDir、eventStore、service control 注入 |
| 修改 | `src/app/bootstrap.js` | 创建并启动/停止 AdminServer，返回 `adminServer`、`runtime`、`attachmentService` 等测试可见依赖 |
| 修改 | `src/index.js` | 启动日志输出 admin URL，保持既有飞书凭据启动逻辑 |
| 修改 | `package.json` | 将新增 `src/admin/*.js` 纳入 `npm run check` 的 `node --check` |
| 修改 | `test/bootstrap.test.js` | 覆盖 admin disabled/enabled 下的 bootstrap 生命周期 |
| 创建 | `test/admin-integration.test.js` | 用真实 HTTP server 组合核心 route，覆盖认证、overview、静态文件和服务关闭 |

high，原因：这是所有模块的生命周期收口点，需要兼顾现有测试、启动关闭顺序、端口隔离和全量检查命令。
