# Task 6: 无构建静态 SPA 控制台

## 描述

实现 `src/admin/public/` 下的原生 HTML/CSS/JS 控制台，覆盖规格中的全部页面和响应式交互。

## 依赖

- 前置 task：T2、T3、T4、T5
- 外部依赖：浏览器原生 Fetch API

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/admin/public/index.html` | 应用壳、登录界面、导航和内容挂载点 |
| 创建 | `src/admin/public/styles.css` | 桌面左侧导航、移动端折叠、表格到紧凑行卡片、危险操作样式 |
| 创建 | `src/admin/public/app.js` | API client、hash 路由、页面渲染、表单提交、确认弹窗、错误展示 |
| 创建 | `test/admin-ui-static.test.js` | 静态资源内容和关键 DOM/API 约束测试 |

## 复杂度

high，原因：单个无构建 SPA 要覆盖多页面、多表单和移动端布局，需要控制文件体积和交互清晰度。

## 实现步骤

- [ ] 实现登录状态检测，token 模式下显示登录表单，登录后通过 bearer 或 cookie 调用 API。
- [ ] 实现页面导航：总览、Sessions、路由绑定、Agent、Runtime、日志、附件、配置、诊断、维护、工具、指标。
- [ ] 为 session 创建、停止、删除、prompt、route 绑定/解绑、配置保存、附件删除、维护清理、服务停止提供表单和确认弹窗。
- [ ] 渲染 events、timeline、metrics、日志、卡片预览和健康检查结果；secret 字段只显示脱敏值。
- [ ] 编写响应式 CSS：桌面左侧导航，390px 宽度下表格切换为紧凑布局，按钮文本不溢出。
- [ ] 编写静态测试，扫描 HTML/JS/CSS 中的关键视图、API endpoint、确认文案和敏感字段保护逻辑。
