# Task 4: 配置编辑、日志、附件、维护与诊断 API

## 描述

实现涉及本地文件系统的管理 API，包含配置读写、日志尾部读取、附件列举下载删除、数据导出备份清理和一键健康检查。

## 依赖

- 前置 task：T1、T2、T3
- 外部依赖：Node.js 内置 `fs`、`path`、`zlib` 可选模块

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/admin/file-admin.js` | 日志读取、附件列举、路径限制、下载和删除 |
| 创建 | `src/admin/diagnostics.js` | 飞书凭据、数据目录、JSON 文件、OpenCode、runtime、日志、孤立 route 检查 |
| 创建 | `src/admin/config-routes.js` | `/api/admin/config` GET/PATCH route 导出 |
| 创建 | `src/admin/maintenance-routes.js` | 日志、附件、导出、备份、清理、诊断 route 导出 |
| 创建 | `test/admin-files-diagnostics.test.js` | 覆盖文件 API、安全路径、健康检查和维护动作 |

## 复杂度

high，原因：文件系统操作必须严格限制目录范围，并且需要在损坏 JSON、缺失目录和清理确认之间保持可恢复响应。

## 实现步骤

- [ ] 实现配置 GET/PATCH route，PATCH 调用 T1 的 allowlist 写入并返回 `restartRequired=true`。
- [ ] 实现日志读取：支持 `out|err`、最近行数上限、关键词过滤、缺失文件空结果和级别过滤。
- [ ] 实现附件列举、下载和删除，所有传入路径必须解析后验证位于附件根目录内。
- [ ] 实现维护导出、备份和清理；清理只处理确认后的 stopped/deleted session route 或孤立附件。
- [ ] 实现一键健康检查，返回 pass/warn/fail 项目数组，页面所需状态不因单项失败而整体 500。
