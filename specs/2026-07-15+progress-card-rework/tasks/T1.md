# Task 1: 调整进度卡片展示模型

## 描述

让 `ProgressCard` 完全忽略最终回答文本，同时保留过程事件展示，并在完成态显示中性的处理完成提示。

## 依赖

- 前置 task：无
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/platform/feishu/progress-card.js` | 调整 text 事件格式化、追加和完成态渲染 |
| 修改 | `test/progress-card.test.js` | 更新旧断言并补充回归测试 |

## 复杂度

low，原因：行为局限于单个卡片模型及其直接单测，不改变公开接口或数据结构。

## 实现步骤

- [ ] 先更新测试，断言 `formatAgentEvent` 对 text 返回空字符串，普通与 delta text 均不进入 `entries` 或卡片元素。
- [ ] 补充 reasoning 长文本仍按 200 字符限制截断、tool_use/error/status 仍按原逻辑展示的测试。
- [ ] 补充 done 后标题模板为绿色、标题为“完成”、正文包含“✅ 处理完成”且不包含先前 text 的测试。
- [ ] 修改 `formatAgentEvent`，使 text 分支返回空字符串。
- [ ] 删除 `append` 中仅服务于 text delta 合并的分支，依赖空格式化结果直接忽略 text，不影响其他事件的阶段更新。
- [ ] 修改 `render`，在 done 状态追加固定中性提示元素，同时保留 done 前已有过程事件。
