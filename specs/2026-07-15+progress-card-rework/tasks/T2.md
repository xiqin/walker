# Task 2: 重构 card 模式最终文本发送

## 描述

让 `MessageDispatcher` 在 card 模式中只用卡片更新过程状态，并由 `_renderEvents` 统一通过 `replyText` 发送一次完整最终回答。

## 依赖

- 前置 task：无
- 外部依赖：现有飞书 API mock 与 `AgentEvent` 类型

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/dispatch/message-dispatcher.js` | 调整 card 渲染、文本发送与 deliveredText 记录条件 |
| 修改 | `test/message-dispatcher.test.js` | 更新 card 断言并补充成功、空值、失败和 watch 补发测试 |

## 复杂度

medium，原因：需协调异步发送返回值、重试失败语义、卡片更新和 session watch 去重，但改动仍限制在单个调度器及其测试。

## 实现步骤

- [ ] 将测试基础 mock 的 `replyText` 成功路径改为返回明确真值结果，避免成功状态被误判为未送达。
- [ ] 更新现有 card 模式测试：TYPE_TEXT 不触发 `updateProgressCard`，合并后的完整回答只通过一次 `replyText` 发送，TYPE_DONE 仍只更新一次。
- [ ] 新增空文本、初始卡片创建失败、`replyText` 返回 `undefined`、`replyText` 重试后失败以及失败后 watchSession 可补发的测试。
- [ ] 在 `_renderCardProgress` 遍历时跳过 TYPE_TEXT，并在无 cardId 时直接返回，不调用 `_renderLegacyProgress`。
- [ ] 在 `_renderEvents` 的 card 分支完成卡片渲染后提取完整文本；仅非空时调用一次 `replyText`。
- [ ] 仅当 card 分支的 `replyText` 返回真值时调用 `_rememberDeliveredText`；失败或无返回值时不记录。
- [ ] 保持 legacy 分支仍由 `_renderLegacyProgress` 发送，并维持现有 deliveredText 记录行为。
- [ ] 确认心跳 TYPE_STATUS、watch 完成转发、doneEmoji 与卡片 patch 失败新消息策略未发生回归。
