# Task 3: turn 生命周期控制

## 描述

在 `MessageDispatcher` 中实现取消、状态查看、最大执行时长和取消/超时后的残留输出防护。

## 依赖

- 前置 task：T1、T2
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/dispatch/message-dispatcher.js` | 增加 `/cancel`、`/status`、`/ps` 处理、turn token、超时看门狗和残留事件抑制。 |
| 修改 | `test/message-dispatcher.test.js` | 覆盖 cancel/status/timeout/重复残留输出。 |

## 复杂度

high，原因：涉及异步 prompt 队列、心跳 timer、watch buffer、session 状态和飞书消息交付去重。

## 实现步骤

- [ ] 步骤 1：在 `handleCommand()` 注册 `cancel`、`status`、`ps`，其中 `ps` 复用 `_cmdStatus()`。
- [ ] 步骤 2：为每个 session 当前 turn 维护轻量运行态，记录 token、开始时间、最近事件时间、progress card、取消状态和超时 timer。
- [ ] 步骤 3：在 `_enqueuePrompt()` 启动 turn 状态和最大执行时长 timer；prompt 完成、失败、取消或超时时统一清理心跳和超时 timer。
- [ ] 步骤 4：实现 `_cmdCancel()`：无绑定回复 `No running session to cancel.`；非 running 回复 `No running turn to cancel.`；running 时调用 driver 的 cancel 或 stop，清理心跳、watch buffer、turn 状态，将 session 标记 `idle` 并回复 `Current turn cancelled: <sessionId>`。
- [ ] 步骤 5：实现 `_cmdStatus()`：无绑定回复 `No session bound to this conversation. Use /new or /attach first.`；有绑定时输出 session id、agent、状态、OpenCode session id、模型、cwd、当前 turn 是否运行、运行时长、最近事件时间和后台 watch 状态。
- [ ] 步骤 6：实现最大执行时长：超过 `maxTurnTimeMins` 时自动执行与 cancel 等价的清理，提示 `Current turn timed out after <N> minutes and was cancelled.`。
- [ ] 步骤 7：扩展重复推送防护：取消或超时后的 prompt 返回事件、watch buffer done、重复 done 不得再次发送最终文本。
- [ ] 步骤 8：补齐异步 timer 测试中的清理逻辑，避免测试进程残留 timer。
