# Task 1: SessionService 1:N routes 升级与迁移

## 描述

将 SessionService 的 routes 数据结构从单值 `{ routeKey: sessionId }` 升级为 `{ focusSessionId, sessions[], cwd }`，新增 addSessionToRoute/setFocus/removeSessionFromRoute/listSessionsInRoute 方法，getCurrent 返回焦点 session，旧格式自动迁移。

## 依赖

- 前置 task：无
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/core/session-service.js` | routes 结构升级、新方法、旧格式迁移 |
| 修改 | `test/session-service.test.js` | 新增 1:N 路由测试用例 |

## 复杂度

medium，原因：数据结构升级涉及多个方法改造和向后兼容迁移，但逻辑边界清晰，无跨模块依赖。

## 实现步骤

- [ ] 步骤 1：新增 `_normalizeRoute(routeKey, state)` 内部方法，将旧单值格式 `state.routes[routeKey] = sessionId` 自动迁移为 `{ focusSessionId: sessionId, sessions: [sessionId], cwd: '' }`。迁移时机：每次 `_ensureState` 后遍历所有 route，检测 `typeof routes[key] === 'string'` 则迁移。
- [ ] 步骤 2：改造 `createSession({ route, agent, title, runtime, cwd, agentRef })`，当 `route` 存在时调用 `addSessionToRoute(route, id, cwd)` 替代直接 `state.routes[route] = id`。
- [ ] 步骤 3：改造 `getCurrent(routeKey)`，从 `state.routes[routeKey].focusSessionId` 读取焦点 session，而非直接 `state.routes[routeKey]`。
- [ ] 步骤 4：改造 `bindRoute(routeKey, sessionId)`，改为调用 `addSessionToRoute(routeKey, sessionId)` 并设为焦点（保持覆盖式语义，向后兼容 `/use` 的旧行为）。
- [ ] 步骤 5：改造 `unbindRoute(routeKey)`，从 route 的 sessions 列表移除焦点 session；若移除后列表为空则删除整个 route 条目；若还有其他 session 则自动切焦点到第一个。
- [ ] 步骤 6：改造 `getRouteForSession(sessionId)`，遍历所有 route 的 `sessions` 数组查找匹配。
- [ ] 步骤 7：改造 `deleteSession(sessionId)`，从所有 route 的 sessions 列表中移除该 sessionId，若为焦点则自动切到下一个活跃 session。
- [ ] 步骤 8：改造 `cleanOrphanRoutes()`，遍历 route 的 sessions 数组清理已删除 session，空列表则删 route 条目。
- [ ] 步骤 9：新增 `addSessionToRoute(routeKey, sessionId, cwd)` 方法：若 route 不存在则创建 `{ focusSessionId: sessionId, sessions: [sessionId], cwd }`；若已存在则 push 到 sessions 数组（去重），不动 focusSessionId（除非是第一个）。
- [ ] 步骤 10：新增 `setFocus(routeKey, sessionId)` 方法：校验 sessionId 在 route 的 sessions 列表中，设置 `focusSessionId = sessionId`，更新 `updatedAt`。
- [ ] 步骤 11：新增 `removeSessionFromRoute(routeKey, sessionId)` 方法：从 sessions 数组移除，若为焦点则自动切到列表第一个，空列表则删 route。
- [ ] 步骤 12：新增 `listSessionsInRoute(routeKey)` 方法：返回 route 的 sessions 列表对应的 session 对象数组（过滤已删除），焦点排在第一位。
- [ ] 步骤 13：新增 `getRouteCwd(routeKey)` 方法：返回 route 的 cwd 字段。
- [ ] 步骤 14：新增 `setRouteCwd(routeKey, cwd)` 方法：设置 route 的 cwd 字段，用于 hook receiver 绑定时记录 cwd。
