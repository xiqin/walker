# Task 6: 心跳轮询与 OpenCode 退出检测

## 描述

为每个 hook 纳入的 session 启动独立心跳轮询，定期检查 OpenCode server 的 `/global/health` 端点。检测到 OpenCode detached 后，取消该 session 的 turn，从 route 的 sessions 列表移除，若是焦点则自动切到下一个活跃 session。

## 依赖

- 前置 task：T1（需要 `removeSessionFromRoute`、`setFocus`）、T2（需要知道哪些 session 是 hook 纳入的）
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 创建 | `src/opencode-hook/health-poller.js` | 心跳轮询管理器 |
| 创建 | `test/opencode-hook-health-poller.test.js` | 心跳轮询测试 |
| 修改 | `src/app/bootstrap.js` | 在 start 中启动 health poller，stop 中停止 |

## 复杂度

medium，原因：定时器管理、多 session 独立心跳、退出时的 cascade 逻辑（取消 turn + 移除 route + 切焦点），需注意 timer 清理和并发安全。

## 实现步骤

- [ ] 步骤 1：创建 `src/opencode-hook/health-poller.js`，导出 `createHealthPoller(options)` 工厂函数，返回 `{ start, stop, track, untrack, getTrackedSessions }`：
  - `options`: `{ sessionService, driverRegistry, dispatcher, pollIntervalMs, exitAction, httpClient }`
  - `track(sessionId, agentRef)`：为指定 session 启动独立轮询定时器，记录到内部 Map
  - `untrack(sessionId)`：停止并移除指定 session 的轮询定时器
  - `start()`：空操作（poller 按 track 驱动）
  - `stop()`：停止所有轮询定时器，清空 Map
  - `getTrackedSessions()`：返回当前被追踪的 sessionId 列表
- [ ] 步骤 2：在 poller 的轮询回调中实现检测逻辑：
  - 每个 session 独立定时器，间隔 `pollIntervalMs`（默认 5000ms）
  - 轮询调用 `httpClient.request('GET', agentRef.serverUrl + '/global/health', null)`
  - 连续 2 次 health 检查失败视为 detached（避免单次网络抖动误判）
  - 检测到 detached 后调用 `_handleDetached(sessionId)`
  - 定时器使用 `setInterval`，每次 `unref()` 避免阻止进程退出
- [ ] 步骤 3：实现 `_handleDetached(sessionId)`：
  - 读取 session 对象，获取 routeKey（通过 `sessionService.getRouteForSession(sessionId)`）
  - 若 `exitAction === 'cancel'` 且 session 有 running turn：调用 dispatcher 的 `_cancelTurn` 取消
  - 调用 `sessionService.removeSessionFromRoute(routeKey, sessionId)` 从 route 移除
  - 调用 `dispatcher._stopSessionWatch(sessionId)` 停止 SSE watch
  - 调用 `poller.untrack(sessionId)` 停止心跳
  - 若 route 为空（removeSessionFromRoute 已删 route），无需额外操作
  - 若 route 有其他 session，`removeSessionFromRoute` 已自动切焦点
  - 日志记录 detached 事件
- [ ] 步骤 4：在 `src/app/bootstrap.js` 的 `createApp` 中创建 health poller 实例：
  - 注入 `sessionService`、`driverRegistry`、`dispatcher`、`pollIntervalMs: config.walkerOpendcodeHealthPollIntervalMs`、`exitAction: config.walkerOpendcodeExitAction`
  - 在 `start()` 中：对已有 hook 纳入的 session 调用 `poller.track(sessionId, agentRef)`（重启恢复）
  - 在 `stop()` 中：调用 `poller.stop()`
  - 将 poller 实例传入 hook receiver（T2 的 receiver 在纳入 session 后调用 `poller.track`）
- [ ] 步骤 5：在 hook receiver 的 `_autoEnrollSession` 成功纳入后调用 `poller.track(walkerSessionId, agentRef)` 启动心跳。需在 receiver 的 ctx 中注入 poller 实例。
