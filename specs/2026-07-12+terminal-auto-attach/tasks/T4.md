# Task 4: Dispatcher 命令改造与非焦点输出

## 描述

改造 MessageDispatcher 的 `/use`、`/list`、`/status` 命令适配 1:N session 路由，实现非焦点 session 的 SSE watch 事件主动回卡片到群里并带 `[session: wks_N]` 标识。在 bootstrap.js 中集成 hook receiver 路由注册和 plugin 安装调用。

## 依赖

- 前置 task：T1（需要 `setFocus`、`listSessionsInRoute`、`removeSessionFromRoute`、`getRouteCwd`）
- 外部依赖：无

## 涉及文件

| 操作 | 文件路径 | 说明 |
|------|----------|------|
| 修改 | `src/dispatch/message-dispatcher.js` | `/use` 改切焦点、`/list` 列 route 下 session、`/status` 显示多 session、非焦点输出回群 |
| 修改 | `test/message-dispatcher.test.js` | 新增 1:N 命令行为和非焦点输出测试 |
| 修改 | `src/app/bootstrap.js` | 集成 hook receiver 路由注册和 plugin 安装调用（注入 receiver 依赖） |

## 复杂度

high，原因：涉及多个命令改造、非焦点 session 输出回群的新逻辑、bootstrap 集成，需确保向后兼容。

## 实现步骤

- [ ] 步骤 1：改造 `_cmdUse(cmd)`，将 `bindRoute(cmd.routeKey, targetId)` 改为 `sessionService.setFocus(cmd.routeKey, targetId)`（切焦点而非覆盖式绑定）。`/use off` 行为改为 `unbindRoute`（从 route 移除焦点 session，保留其他 session）。校验 targetId 在 route 的 sessions 列表中，不在列表则返回错误。
- [ ] 步骤 2：改造 `_cmdList(cmd)`，从 `listSessions()` 改为 `listSessionsInRoute(cmd.routeKey)`（只列出当前 route 下的 session，而非全局所有 session）。标记焦点 session。
- [ ] 步骤 3：改造 `_cmdStatus(cmd)`，显示 route 下多 session 状态。格式：
  ```
  Route: <routeKey> (cwd: <cwd>)
    Active sessions: <N>
    Focus: <focusSessionId> (<agent>, <status>, <opencodeSessionId>)
    Other: [<sessionId> (<status>), ...]
  ```
  无 session 时仍显示"未绑定"提示。
- [ ] 步骤 4：改造 `_handleWatchedSessionEvent(session, chatId, agentEvent)`，在发送文本到群时加上 session 标识前缀：`[session: <sessionId前8位>] <text>`。判断当前 session 是否为 route 的焦点：是焦点则不加前缀（保持原有体验），非焦点则加前缀。需读取 `sessionService.getCurrent(session.route)` 判断是否焦点。
- [ ] 步骤 5：新增 `nonFocusOutput` 配置项注入到 MessageDispatcher 构造参数。`nonFocusOutput === false` 时，非焦点 session 的 `_handleWatchedSessionEvent` 直接 return（静默），不回群。
- [ ] 步骤 6：改造 `handleIncomingMessage`，`getCurrent(routeKey)` 返回焦点 session 的行为不变（T1 已保证），无需额外改动，但需确认 `_ensureWatch` 对非焦点 session 也启动 watch（因为 hook 纳入的非焦点 session 也需要 watch 以回群）。
- [ ] 步骤 7：在 `src/app/bootstrap.js` 的 `createApp` 中：
  - 注入 `nonFocusOutput: config.walkerOpendcodeNonFocusOutput !== false` 到 MessageDispatcher 构造参数
  - 在 `start()` 中调用 `installHookPlugin({ opencodeConfigDir, walkerPort: adminConfig.port, enabled: config.walkerOpendcodeHookEnabled })`
  - 将 hook receiver 路由注册到 admin server（在 `createAdminServerFromContext` 调用时传入 hook receiver 路由数组）
- [ ] 步骤 8：改造 `_cmdCancel(cmd)`，取消焦点 session 的 turn。行为不变（`getCurrent` 返回焦点），但需确认在 1:N 场景下正确取消焦点而非误取消其他 session。
