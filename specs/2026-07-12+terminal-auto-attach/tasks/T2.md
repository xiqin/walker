# Task 2: Hook plugin 安装与 receiver 端点

## 描述

实现 Walker 启动时自动写入 hook plugin 到 `~/.config/opencode/plugins/walker-hook.js`（不覆盖已存在文件），新增 HTTP 端点 `POST /opencode/hook/session-created` 接收 plugin 上报 `{ opencodeBaseUrl, sessionId, cwd }`，按 cwd 匹配 routeKey 创建 Walker session 并加入 route 的 1:N sessions 列表。只接受 loopback 请求，复用 admin token 鉴权。

## 依赖

- 前置 task：T1（需要 `addSessionToRoute`、`setRouteCwd`、`getRouteCwd` 等新方法）
- 外部依赖：无

## 实现步骤

- [ ] 步骤 1：创建 `src/opencode-hook/plugin-template.js`，导出 `getPluginSource(baseUrl)` 函数，返回 plugin JS 文件内容字符串。Plugin 逻辑：
  - 监听 `session.created` 事件
  - 提取 `session.id` 和 `session.cwd`（或从 plugin context 的 `directory` 获取 cwd）
  - POST 到 `http://127.0.0.1:<port>/opencode/hook/session-created`，body 为 `{ opencodeBaseUrl, sessionId, cwd }`
  - Walker 不可达时静默忽略（catch 网络错误，不抛出）
  - 硬编码 Walker 地址为 loopback，不包含 token 等敏感信息
- [ ] 步骤 2：创建 `src/opencode-hook/installer.js`，导出 `installHookPlugin({ opencodeConfigDir, walkerPort, enabled })` 函数：
  - `enabled === false` 时直接返回 `{ installed: false, reason: 'disabled' }`
  - 检查 `~/.config/opencode/plugins/walker-hook.js` 是否存在
  - 已存在则返回 `{ installed: false, reason: 'already_exists' }`（不覆盖）
  - 不存在则写入 plugin 文件内容，返回 `{ installed: true, path }`
  - 使用 `os.homedir()` 拼接 `.config/opencode/plugins/walker-hook.js`
- [ ] 步骤 3：创建 `src/opencode-hook/receiver.js`，导出 `createHookReceiverRoutes(ctx)` 函数，返回路由数组：
  - `POST /opencode/hook/session-created`：解析 body `{ opencodeBaseUrl, sessionId, cwd }`
  - 校验：`sessionId` 和 `cwd` 非空，否则返回 400
  - loopback 校验：检查请求来源 IP 是否为 `127.0.0.1` 或 `::1`，否则返回 403
  - admin token 复用：通过 `isAuthenticated(req, config)` 校验（admin token 未配置时放行）
  - 调用 `_autoEnrollSession(ctx, { opencodeBaseUrl, sessionId, cwd })` 纳入 session
  - 返回 `{ ok: true, sessionId: walkerSessionId, routeKey }`
- [ ] 步骤 4：在 receiver 中实现 `_autoEnrollSession(ctx, { opencodeBaseUrl, sessionId, cwd })`：
  - 检查是否已有 Walker session 持有相同 `agentRef.opencodeSessionId`（避免重复纳入）
  - 已存在则返回该 session（幂等）
  - 按 cwd 找 routeKey：遍历所有 route，匹配 `route.cwd`（精确优先，子目录次之）
  - 多候选取最近活跃（focusSessionId 对应 session 的 updatedAt 最大）
  - 找到 routeKey：创建 Walker session（agent='opencode', cwd, agentRef={ opencodeSessionId, serverUrl: opencodeBaseUrl }），调用 `addSessionToRoute(routeKey, sessionId, cwd)`
  - 未找到 routeKey：创建游离 session（不绑 route），返回 `{ sessionId, routeKey: null }`
- [ ] 步骤 5：在 `src/app/bootstrap.js` 的 `start()` 函数中调用 `installHookPlugin`（读取配置 `WALKER_OPENCODE_HOOK_ENABLED` 和 admin port）。此步骤仅添加调用，不 owns bootstrap.js（T4 负责 bootstrap 的其他改动）。
